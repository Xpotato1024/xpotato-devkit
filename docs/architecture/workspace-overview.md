# xpotato-devkit アーキテクチャ

本ドキュメントでは、`xpotato-devkit` の Rust バックエンドのモジュール構成（Workspace）について解説します。

## 全体構成

このツールは Cargo workspace 機能を用いて、役割ごとに複数の Crate（クレート）に分割されています。
これによりコンパイル単位を適切に分離し、開発時の見通しとテスト性を向上させています。

```text
rust/
  Cargo.toml (Workspace root)
  crates/
    devkit-cli/      # ユーザーインターフェース (CLI) 層
    devkit-core/     # ツールキットの共通型と基本機能
    devkit-block/    # プログラミング言語やMarkdownの構造解釈 (block抽出など)
    devkit-git/      # Git リポジトリ操作
    devkit-...       # 今後の拡張に応じたプラグイン的構成
```

## 各 Crate の役割

### `devkit-cli`
最終的な実行ファイル（バイナリ）を生成するアプリケーション層です。
- コマンドライン引数のパース (`clap` を利用)
- サブコマンドのディスパッチ
- 出力のフォーマットや標準出力への書き込み
- `--brief` などの表示制御ロジック

### `devkit-core`
他すべてのクレートが依存できる共通ライブラリです。
- エラーハンドリング基盤
- 設定ファイル (`devkit.toml`) の読み込みとパース
- プロジェクトの言語判定や拡張子マッピングなど、全般的なユーティリティ

### `devkit-block`
長大なテキストデータを「関数」や「Markdown サブセクション」といった意味のある「ブロック」単位で扱うロジックを担います。
- ツリーシッター(`tree-sitter`) を用いたソースコードのAST解析
- MarkdownのAST探索や置換機能
- インデントベース (Python等) のフォールバック解析
- `extract`, `replace`, `outline` コマンド向けの中核機能

### `devkit-git`
Git に関するコマンド（`diff`, `status`, コミット情報取得など）を担うモジュールです。
- ローカルリポジトリへの安全なアクセス
- 差分統計の集計
- AI向けプロンプト (Commit / PR用) の抽出機能

## 将来の拡張方針

新規に独立した機能 (例: ASTを用いた大規模なリファクタリング支援など) が必要になった場合は、`devkit-core` に全てを詰め込むのではなく、新規クレート (`devkit-xxx`) として追加することを推奨します。
