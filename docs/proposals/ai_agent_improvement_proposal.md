# devkit AI エージェント改善提案

## 目的

`devkit` は、エージェント主導のワークフローに適した基盤になっています。diff の要約、block の抽出、patch の適用、git 補助のような小さく合成しやすいコマンドを提供しており、人間にも自動化エージェントにも扱いやすい構成です。

この提案は、repo 非依存かつクロスプラットフォームという前提を保ちながら、AI がより推論しやすい道具にするための次の改善案をまとめたものです。

## すでに良い点

- コマンドが狭く、単一目的に絞られています。
- 出力の多くが決定的で、スクリプトから扱いやすいです。
- 抽出、置換、patch、要約の責務が分かれています。
- CI ですでにフォーマットと静的解析を検証しています。

## AI 利用での課題

1. コマンド間で出力契約が完全には統一されていません。
2. 失敗時の報告は人間には有用ですが、自動再試行には必ずしも最適ではありません。
3. 一部のコマンドは、機械可読なモードよりも人間向けの文言が前面に出ています。
4. ドキュメントは機能説明としては十分ですが、推奨エージェントワークフロー全体はまだまとまっていません。

## 提案

### 1. コマンド出力を標準化する

- 実用上可能な範囲で、統一された `--json` モードを追加します。
- `--brief` は人間がざっと確認する用途に残しつつ、コマンド群全体で構造を予測しやすくします。
- エラーは安定した接頭辞と、安定した終了コード方針で扱います。

### 2. patch-first ワークフローを改善する

- `patch diagnose` と `patch apply` を連携しやすくします。
- 可能な限り、最小単位で行動可能な失敗情報を出します。
- dry-run の出力と reject 時の出力を、機械が解析しやすい形に保ちます。

### 3. block 操作をより明示的にする

- block の抽出・置換ガイドを拡充し、エージェントが適切な selector を素早く選べるようにします。
- 複数の selector が与えられたときの優先順位を明確にします。
- 行範囲、見出し、マーカー、シンボルによる抽出の挙動を、1 か所で読めるようにします。

### 4. エージェント向けドキュメントを追加する

- 推奨手順を示す短いワークフローガイドを追加します。
  - diff を要約する
  - 対象 block を確認する
  - patch を適用する
  - 結果を検証する
- どのコマンドが計画、編集、検証に向いているかを明記します。

### 5. 検証手順をエージェント用途に合わせる

- `fmt`、`clippy`、対象を絞ったテストを第一級の検証として扱います。
- 各コマンドの近くに、対応するローカル検証コマンドを記載します。
- 文章だけの保証よりも、決定的なチェックを優先します。

## 推奨フェーズ

### Phase 1: 出力契約の整理

安定した出力形式、一貫したエラー処理、予測しやすい brief モードに集中します。

### Phase 2: block と patch の操作性改善

block selector、patch の診断、安全な編集フローに集中します。

### Phase 3: エージェントワークフローの文書化

`diff`、`block`、`patch`、`git` をどう組み合わせるべきかを示す、簡潔なガイドを 1 つ作成します。

### Phase 4: 検証と保守

コマンドの進化に合わせて、CI、ローカルチェック、ドキュメントを同期させます。

## 成功条件

- エージェントが、コマンドの意味を推測せずに、変更を把握し、正しい block を確認し、編集を適用し、結果を検証できること。
- 人間の利用者に対しても、出力が簡潔であること。
- コマンドの挙動が repo 非依存で、Windows / Linux / WSL のどれでも予測可能であること。
- ドキュメントが、推奨ワークフローと失敗時の扱いを明確に説明していること。

## 非目標

- リポジトリ固有の前提は追加しません。
- 対話専用のフローは導入しません。
- このツールを検索システムやランキングシステムに拡張しません。
- CLI を特定の下流プロジェクトのレイアウトに依存させません。

## まとめ

`devkit` の次の改善は、機能をむやみに増やすことではありません。AI エージェント向けの運用契約をより厳密にすることです。具体的には、一貫した出力、明示的な失敗モード、そして `inspect -> edit -> verify` の流れを 1 本に揃えることです。
