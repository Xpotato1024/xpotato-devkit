# Rust移行検証テスト結果レポート

`docs/devkit_rust_migration_test_strategy.md` に基づき、`xpotato-devkit` の主要コマンド群について複数回のベンチマークテストを実施しました。本レポートではその結果と、Rust移行に関する結論をまとめます。

## 計測環境

- **OS/環境**: Windows (`.venv\Scripts\devkit.exe` の直接呼び出し)
- **計測回数**: ウォームアップ各2回、本計測各20回
- **計測対象プロジェクト**: `xpotato-devkit` 自体（小〜中規模クラス）

## 計測結果サマリ（抜粋）

計測された結果（単位はミリ秒、平均値）は以下の通りです。

| シナリオ | total (wall) | startup | git | I/O | logic |
|:---|:---:|:---:|:---:|:---:|:---:|
| **A1_tree_depth3** (`devkit tree --max-depth 3`) | **291.8** | 291.3 | 0.0 | 0.0 | 0.5 |
| **B1_outline** (`devkit block outline ...`) | **253.8** | 251.6 | 0.0 | 0.2 | 2.0 |
| **B2_context** (`devkit block context ...`) | **245.7** | 243.2 | 0.0 | 0.2 | 2.3 |
| **C1_extract** (`devkit block extract ...`) | **254.2** | 251.9 | 0.0 | 0.2 | 2.0 |
| **D1_impl_note** (`devkit doc impl-note ...`) | **284.0** | 245.7 | 37.3 | 0.0 | 1.0 |
| **E1_patch_diag** (`devkit patch diagnose test.patch`) | **297.1** | 296.6 | 0.0 | 0.0 | 0.5 |

## 分析

1. **Python 起動コストが圧倒的に支配的**
   - ほぼ全てのコマンドにおいて、全実行時間（`wall_ms`）の **98%〜99% が Python の起動・モジュール初期化コスト**（`startup_ms` ≒ 250〜300ms）によって占められています。
   - `typer`, `rich`, その他構造解析を含むモジュールの import や実行環境立ち上げに固定で 250ms 以上かかっています。

2. **内部ロジック自体は極めて高速**
   - 構造解析（`outline`, `context`, `extract`）やパッチ診断、ファイルツリー検索などの Python で実装された内部アルゴリズムコスト (`logic_ms`) は、それぞれ **1〜3ms 程度** に収まっています。
   - すなわち、ファイル I/O と探索アルゴリズムは十分に最適化されており、これら単体を Rust 化しても短縮できる時間は 1〜3ms しかありません。

3. **git 呼び出しコストの影響は軽微**
   - 唯一 git を呼び出すシナリオ (`impl-note`) では git subprocess に 37ms 消費していますが、それでも全体 (284ms) に占める割合は 15% 未満に過ぎません。

## 結論と推奨アクション

方針書に記載された判断基準に照らし合わせると、現在のボトルネック構成は次のように判定できます。

> **10.2 全面移行を検討すべき条件に合致**
> - 「高頻度コマンド群の多くで Python 起動コストが支配的」
> - 「部分移行では効果が分散しすぎる」

内部ロジックがいかに高速化されても、Python ランタイム自体の起動コスト（約 250ms〜300ms）の影響で、一連の開発タスクの中で高頻度かつ短いコマンド（AIエージェントによる連続的なアウトライン探索など）を叩く場合、大きな体感遅延が発生します。

**結論：【 C. Rust 本格移行 】を強く推奨します。**

ハイブリッド化（一部を Rust 化して PyO3 等でバインディング）では、Python の起動制約を取り除けません。CLI 全体を Rust で再実装し、完全なネイティブバイナリとして配布・実行可能にすることで、上記の 250ms オーバーヘッドを完全に削ぎ落とし、全コマンドの応答時間を 10ms〜50ms 以内に劇的改善できる見込みです。
